Q. 1) I can see that one (or many) songs aren't identified by my application. Why is this so?

There can be multiple reasons for this. Our application will only recognize MP3 files. WAV, FLAC, AAC, etc. files will not be recognized. As long as your song file has an extension of mp3 (or MP3), run this quick check list and see if your song meets at least one of the following criteria:

I) Your music file has a valid ID3 tag.
II) You are following the Artist - Title naming convention for your music file name.
III) Your music file is present in the SD card. Our application will only scan the SD card for music, so please make sure that you migrate all your songs to the SD card. Placement is irrelevant.
IV) Your music file is not hidden.

Q. 2) Why does querying song information take so much time?

If your music is large (and valid), querying will probably take some time. We've found that it takes around 1 to 1.25 minutes to query a batch of 100 valid songs. We'd recommend using a 3G/4G connection or WiFi.

Q. 3) Is it mandatory for my MP3 files to have an ID3 tag?

Not really, but we'd highly recommend it. Our song filtering algorithm will first check ID3 tag validity and then move on to the file name in case of failure of the former. However, if the file name validity check fails, the song is discarded.

Q. 4) What is this 'file name validity'?

As mentioned in Ans. 3, our algorithm goes through two stages of file filtering - the ID3 tag stage and the file name stage. If our algorithm does not detect a valid ID3 tag, it checks the name of the file. The file name MUST be in the format "Artist - Title" (without quotes).
E.g.: Pink Floyd - Coming Back to Life.mp3

Yes, the application development is still in it's alpha stage. Sure, you can contribute! Visit this GitHub page and follow the contribution procedure in the Readme.

Q. 6) How do you guys retrieve the song mood?

We use a state-of-the-art audio fingerprinting mechanism that allows us to retrieve audio moods via an acoustic neural network. Just kidding, we use the Gracenote API. You can check out their developer website here.

Q. 7) Which libraries have you used for this application?

You can find the complete list of dependencies on the GitHub link posted in Ans. 5.

Q. 8) How is the music mood mapped to my facial mood?

That's where the genius of the application (and the developers) lies. If you visit the GitHub page, you'll see three diagrams on the front page. The first one tells you how Gracenote categorizes moods. The second is the 5x5 mood matrix, also called the Level one mood. The third image is a 10x10 matrix, a.k.a. the Level Two Mood. The Level Two mood is a fine-grain version of the Level One mood. We map your smiling probability to the 5x5 and 10x10 matrix. If you need further clarification, please let us know so.
